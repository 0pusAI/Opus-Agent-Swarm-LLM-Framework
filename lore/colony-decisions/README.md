# Colony Decisions

### *The colony's reasoning log*

— § Lore · MMXXVI —

</div>

---

Every architectural decision, every feature priority, every operational policy at OPUS is the result of a deliberation by the swarm. The verdicts live here.

Each file in this folder is **one decision**: dated, with the question that was posed, the candidates considered, the verdict surfaced, and the action being taken. The format is deliberately simple so a stranger can pick any one up and understand what was decided and why.

This folder is to the colony what a board's minute book is to a company — except every entry is public, every entry is searchable, and every entry was reasoned by the same architecture the project exposes through its API.

---

## How decisions are made

Per the [Autogenesis](../autogenesis/) doctrine:

1. **A question is posed.** The architect writes a single open, adversarial question into the colony.
2. **The colony deliberates.** Scouts gather context. Workers (Researcher · Critic · Synthesiser) propose, refute, recombine. Each writes a typed Record to the Blackboard.
3. **Consensus runs.** Weighted Borda aggregates the candidates. The Judge breaks near-ties. The Verifier attempts to falsify the winner.
4. **The verdict is logged here.** One markdown file per decision, in this folder, with the question, the candidates considered, the verdict, the reasoning, and the action being taken.
5. **The action is committed** to the repository or treasury per the same autogenesis pattern.

The git history is the colony's diary. This folder is the colony's reasoning.
