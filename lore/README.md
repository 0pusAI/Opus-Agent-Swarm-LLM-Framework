# `lore/`

*Artifacts of OPUS that live outside the codebase but belong to the work.*

</div>

---

This folder is not required to build, run, or understand the OPUS swarm engine. None of it is imported by `opus-core/`. None of it is rendered by `opus-web/`. The project would still function without it.

It would also be smaller without it.

`lore/` is where the parts of OPUS that are *cultural* rather than *technical* live — the visual codex, the architect's notes, the long-form lineage essays, the marginalia that the documentation proper would refuse. Treat it as a scriptorium attached to the workshop: every artefact in here was made carefully, by hand, with the same constraints that govern the system itself.

---

## Contents

- **[`autogenesis/`](autogenesis/)** — *the doctrine at the centre of the project.* OPUS was not built top-down by a single mind; it was — and continues to be — built by the swarm it is. The Ouroboros principle, the workflow by which every feature is born inside the colony, and the meta-sigil that contains the sixteen daily ones. Read this first.
- **[`sigils/`](sigils/)** — the sixteen daily build-log artworks, reframed as a closed alchemical codex with a long-form thesis, a glossary of the recurring visual grammar, and an interpretive note for each piece. Includes the SVG sources, a bundled `opus-sigils.zip`, and a full [`README.md`](sigils/README.md) that renders as the canonical sigils document on GitHub.

*More to come.*
